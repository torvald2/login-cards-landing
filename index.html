NFT as passwordless identity

Features

  • OAUTH2 implicit and id token flow support
  • Yor custom claims stored in the token`s metadata that stores in IPFS and are unchangeable
  • Login cards provider uses WEB3 passwordless technologies for users authorisation and check NFTs owner before issue JWT

How it works

  1. User put "Sign up button" on your app
  2. App redirecs user to the trusted.cards login page
  3. User connects his wallet. login.cards creates new user and ask him sign some one time nonce. If user does not have login.card nft for your app we will mint it for user.
  4. Login.card mint token in the Polygon chain and store metadata on the IPFS. Transaction fee for the mint optional - from users or app account wallet
  5. Eatch login login.cards change NFT owner
  6. And issue JWT ID/Reflash token where custom claims mapped from the NFT
  7. Your app can use JWT and validate signature via JWKS url
